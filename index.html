Program Editor (New Admin File)
New Feature

File provided: happytask-admin-dev-update.html ⬇ Download File

This file was built directly from your live admin source code pulled from GitHub. Your code is untouched. We only added three things on top:

  • Group Settings panel removed
  • Day filter dropdown (replaces Group dropdown)
  • Program Editor panel + all its JavaScript

Watch this 60-second walkthrough to see it in action:

The Program Editor lets admins edit daily tasks for all 14 days — add/remove tasks, edit descriptions, build guide content with rich blocks (headings, templates, step buttons, Q&A, warnings), reorder with drag-and-drop, preview in participant view, and publish live.

What still needs to be wired up:

1
Server endpoint to store published configs
When an admin clicks "Publish," the Program Editor sends the updated task config as a JSON POST. You need a server endpoint that saves this and serves it back on GET. How you store it (file, database, etc.) is up to you.
2
Participant dashboard reads the published config on load
When a participant opens their dashboard, it should fetch the published config from that same endpoint and use it to override the default task data. If the server is unreachable, fall back to a cached version in localStorage so the dashboard still works offline.
